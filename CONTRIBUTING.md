# Contributing to Atlas Protocol

We appreciate your interest in contributing! This guide will help you get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/atlas-protocol.git`
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Install dependencies: `npm install`
5. Run dev server: `npm run dev`

## Development Guidelines

### Code Style
- Use TypeScript for all new code
- Follow existing code patterns and conventions
- Use meaningful variable and function names
- Add JSDoc comments for complex functions

### Components
- Keep components focused and reusable
- Use shadcn/ui components when possible
- Implement proper error handling
- Add loading states for async operations

### Testing
- Write tests for new features
- Ensure all tests pass: `npm run test`
- Aim for >80% code coverage

### Commits
- Use descriptive commit messages
- Reference related issues: `Fix: wallet connection issue (#123)`
- Keep commits atomic and logical

## Pull Request Process

1. Ensure your code passes linting: `npm run lint`
2. Ensure build succeeds: `npm run build`
3. Write a clear PR description explaining changes
4. Link related issues
5. Wait for review and address feedback
6. Squash commits if requested

## Reporting Bugs

1. Check if bug is already reported
2. Include steps to reproduce
3. Include expected and actual behavior
4. Include browser/environment details
5. Attach screenshots if relevant

## Feature Requests

1. Check if feature is already requested
2. Clearly describe the feature
3. Explain the use case
4. Include mockups/examples if possible

## Questions?

- Open an issue with the `question` label
- Check existing documentation
- Review previous issues and discussions

Thank you for contributing! 🎉
